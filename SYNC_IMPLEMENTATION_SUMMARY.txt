CODE INTEGRATION GUIDE
================================================================================

BACKEND INTEGRATION

Add to request pipeline (after RBACMiddleware):
  use App\API\Middleware\EnhancedRBACMiddleware;
  
  // In controller before action
  $permissions = EnhancedRBACMiddleware::resolvePermissionsWithContext(
    $userId,
    $workflowId,    // optional
    $stageId        // optional
  );

Check route access:
  if (!EnhancedRBACMiddleware::canAccessRoute($userId, $routeName)) {
    return $this->forbidden('No access to this route');
  }

Get data scope:
  $scope = EnhancedRBACMiddleware::getUserDataScope($userId);
  if ($scope['data_level'] === 'full') {
    // Show all data
  } else if ($scope['data_level'] === 'school') {
    // Filter to school-wide data
  }

FRONTEND INTEGRATION

Include in page:
  <script src="/js/components/EnhancedRoleBasedUI.js"></script>

Guard entire component:
  <div data-module-permission data-module="finance" data-action="approve">
    <!-- Only shown if user has finance_approve permission -->
  </div>

Guard individual actions:
  <button id="submit-approval" data-guard-action="finance_approve">
    Submit for Approval
  </button>

Guard workflow stages:
  <div data-workflow="FEE_APPROVAL" data-workflow-stage="approval" 
       data-stage-actions="finance_approve">
    <!-- Only shown if in FEE_APPROVAL workflow at approval stage -->
  </div>

Check in JavaScript:
  if (EnhancedRoleBasedUI.hasModulePermission('finance', 'approve')) {
    // Enable finance approval UI
  }
